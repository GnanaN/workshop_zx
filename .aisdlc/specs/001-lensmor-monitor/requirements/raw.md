Lensmor Monitor 是一个自动化的竞品情报监控平台。它帮助团队持续追踪竞争对手的动态——包括网站改版、社交媒体的讨论热度等，并用自然语言告诉你：发生了什么变化、为什么值得关注、你下一步该怎么做。

核心价值主张是：把过去需要手动浏览、截图、人工比对的竞品调研工作，变为一个只需设定监控目标、定期收取结构化情报卡片的过程。

目标用户：
- 产品经理：关注竞品的功能迭代、产品策略变动，需要每周了解对手产品方向
- 市场营销负责人：关注竞品的定价变化、营销活动、社媒风评，用于调整自身市场策略
- 创业者 / 创始人：资源有限，需要高效了解赛道内主要对手的动向，辅助商业决策
- 投资人：需要持续跟进被投公司所在赛道的竞争格局变化

功能需求：
- 新手引导
- 竞品监控管理
- 竞品详情
- 情报邮件箱

## 详细功能需求（用户补充）

### 3.1 新用户引导
用户首次进入系统时，需要完成一个 3 步引导流程：
- 第一步：选择角色（产品营销经理、产品经理、市场营销经理、创始人、投资人、其他）。角色影响情报解读角度。
- 第二步：填写自有产品信息（名称、URL、一句话描述、目标受众、核心卖点、竞争优势、当前阶段战略目标）。用于系统判断情报相关性和重要性。
- 第三步：导入竞品。系统根据产品 URL 自动推荐潜在竞争对手和相关品牌关键词。用户可勾选或手动添加。
- 补充能力：系统根据用户产品 URL 自动发现相关子域名、定价页面、产品介绍页等关联链接，纳入监控范围。

### 3.2 竞品监控管理
- 竞品列表左侧边栏展示：Logo、名称、当前状态（监控中/已暂停/采集进行中）
- 支持添加、编辑、删除竞品
- 添加时可额外补充关联链接（上限 10 条），如子品牌站、开发者文档站等
- 暂停/恢复监控：暂停后停止定时采集，可随时恢复
- 手动刷新：随时触发全量刷新，强制重新采集并生成最新报告

### 3.3 竞品详情
点击竞品后右侧展示综合概况。Overview 面板（默认视图）：
- 公司基本面：成立时间、公司类型、融资阶段、所在地、员工规模、目标细分市场、领英主页等
- 最新情报：时间线卡片形式展示近期重要变化

### 3.4 情报收件箱
核心模块。**情报收件箱是系统内模块（非真实邮箱）。**
两类收件箱：
- 分析报告 (Analysis Report)：基于网站变化的自动化分析结果
- 战略总结 (Strategic Summary)：基于多条分析报告的综合性分析

交互：左侧卡片列表 + 右侧详情。卡片展示标题、关联竞品、时间、优先级标签（紧急/中等/低影响）。支持按优先级、日期范围、竞品筛选。打开即标记已读。

情报详情内容：
- 网站变化分析报告：变更摘要列表、战略意图分析、行动建议（按紧急程度分级，附理由）、原始网站跳转链接
- 历史差异分析：90 天演变分析，多时间点逐对变化对比，标注变化时间+类型（营销/运营/功能/定价），整体战略演进摘要
- 战略总结：跨报告综合研判 + 行动建议计划

### 3.5 情报反馈
用户对每条情报提交反馈：
- 有用 (Useful)
- 有误 (Wrong)：可选错误类型（信息不准确、内容不相关、数据过时、信息重复、战略意图有误、遗漏关键变更、噪音过多、源数据不准确）
- 不重要 (Not Important)

### 4. 数据采集与分析引擎
#### 4.1 数据采集通道
- **网站内容采集**：竞品网站当前页面内容（文字 + 截图）
- **历史快照采集**：竞品网站的历史版本（回溯数月到数年）
- **社媒平台采集**：以 Reddit 为主的社交媒体帖子和评论
- **关键词/竞品情报**：关键词搜索量、付费/自然流量竞品排名

> MVP 范围：仅网站内容采集。历史快照、社媒、关键词采集为 M1 后续迭代。

#### 4.2 AI 分析管道
- 网站差异分析：前后对比，识别有意义的变更
- 历史演变分析：多时间点快照串联，分析长期演进路线
- 社媒品牌分析：情绪基调、关键洞察和风险信号
- 公司基本面分析：结构化提取竞品基本属性
- 战略总结：汇聚多条原子分析报告，输出综合性战略判断
- 行动建议生成：基于分析结果，生成可执行应对策略

#### 4.3 数据清洗
HTML→Markdown 转换、文本去噪、内容摘要提取、语义级差异标注

### 5. 系统运行机制
异步任务调度体系：任务队列支持排队、重试、取消、暂停。定时监控按配置间隔自动执行周期采集和分析。

### 6. 技术要求
- 性能：首屏加载 ≤3s，列表分页加载 ≤2s，后台任务并发处理
- 可用性：异步操作加载状态、空数据引导、异常友好提示
- 安全：登录认证、用户数据隔离、签名链接访问敏感资源、定时任务端点密钥保护

## 澄清记录

### 第 1 轮澄清
- 本轮结论：产品形态为 **Web 平台**。情报收件箱是系统内模块，非真实邮箱。
- 关键决策：产品形态 → Web Dashboard 为主（对应选项 A）

### 第 2 轮澄清
- 本轮结论：MVP 范围为 **核心情报闭环**（选项 A）。
- MVP 包含：3 步引导 + 竞品管理(CRUD/暂停/手动刷新) + 竞品详情 Overview + 分析报告收件箱(含详情) + 情报反馈 + 网站差异分析管道 + 异步任务队列
- MVP 不包含（后续迭代）：历史差异分析(90天)、战略总结、社媒分析、公司基本面自动提取
- 已知缺口：4.1 数据采集通道详情暂无法展示，将在 solution.md 中标记为 CONTEXT GAP

### 第 3 轮澄清（C2 补充）
- 本轮结论：4.1 数据采集通道共 4 项——网站内容采集、历史快照采集、社媒平台采集(Reddit)、关键词/竞品情报。MVP 仅网站内容采集，其余为 M1。
- 关键决策：MVP 采集范围仅 1 项（网站内容），不改变推荐架构
