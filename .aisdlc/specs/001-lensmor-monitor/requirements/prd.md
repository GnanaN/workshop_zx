## 1. 结论摘要

- **目标**：为 PM、市场负责人、创始人、投资人提供一个自动化竞品情报监控 Web 平台，替代手动竞品调研。
- **In / Out 边界**：In = 核心情报闭环（引导→监控→采集→分析→收件箱→反馈）；Out = 历史差异分析(90天)、战略总结、社媒分析、公司基本面自动提取。
- **MVP 边界**：网站差异分析单次报告闭环；竞品发现优先手动添加；不做实时推送通知。
- **推荐方案**：Web Dashboard 单体应用 + 异步任务队列（详见 `requirements/solution.md#2`）。
- **优先验证点**：V-001（竞品发现数据源）、V-002（差异分析质量）、V-003（采集性能瓶颈）。

## 2. 范围与里程碑

### 2.1 MVP 范围（In / Out）

- **In**：
  - 3 步新用户引导（角色选择 → 产品信息 → 导入竞品）
  - 竞品管理（CRUD + 暂停/恢复 + 手动刷新），左侧边栏展示
  - 竞品详情 Overview 面板（基本面 + 最新情报时间线）
  - 分析报告收件箱（卡片列表 + 详情，筛选/已读）
  - 情报反馈（有用 / 有误 / 不重要）
  - 网站差异分析管道（采集 → 清洗 → 差异分析 → 报告生成）
  - 异步任务队列（排队/重试/取消/暂停）
  - 用户认证（JWT + 邮箱密码登录）
  - 用户数据隔离
- **Out**：
  - 历史差异分析（90 天串联）
  - 战略总结（跨报告综合研判）
  - 社媒品牌分析
  - 公司基本面自动提取（MVP 允许手动填写或简单抓取）
  - 实时推送通知（邮件/Slack/IM）
  - SSO / 第三方登录

### 2.2 里程碑

- **MVP**：核心情报闭环可用。用户可完成引导→添加竞品→系统自动采集并生成分析报告→用户在收件箱查看并反馈。
- **M1（后续）**：历史差异分析 + 战略总结 + 社媒分析 + 公司基本面自动提取。

## 3. 核心场景（3 个）

### 3.1 场景 S-001：新用户上手并获取第一份情报

- **触发**：新用户首次访问系统
- **参与者**：产品经理（或其他角色）
- **目标**：完成引导流程 → 添加至少 1 个竞品 → 系统自动采集并生成第一份分析报告
- **成功标准**：
  1. 用户在 5 分钟内完成 3 步引导并成功添加首个竞品
  2. 系统在竞品添加后自动触发首次采集，采集完成后生成分析报告推送至收件箱
  3. 用户可在收件箱看到报告卡片并打开阅读详情

### 3.2 场景 S-002：日常情报消费与反馈

- **触发**：用户定期登录查看竞品动态
- **参与者**：产品经理 / 市场负责人 / 创始人
- **目标**：快速浏览收件箱 → 筛选高优先级情报 → 阅读详情 → 提交反馈
- **成功标准**：
  1. 收件箱卡片列表 ≤2s 加载，支持按优先级/日期/竞品筛选
  2. 点击卡片后在 ≤3s 内展示完整详情（变更摘要 + 战略意图 + 行动建议）
  3. 用户可提交反馈（有用/有误/不重要），提交后即时确认

### 3.3 场景 S-003：竞品生命周期管理

- **触发**：用户需要调整监控范围
- **参与者**：产品经理 / 市场负责人
- **目标**：添加新竞品 → 查看竞品概况 → 暂停不重要的竞品 → 恢复监控或删除
- **成功标准**：
  1. 添加竞品（填写名称+URL+关联链接）后即刻出现在左侧列表
  2. 暂停竞品后状态更新为"已暂停"，该竞品不再触发新采集
  3. 手动刷新触发全量重新采集，状态变为"采集进行中"

## 5. 业务规则与口径

- **规则-1 角色影响解读角度**：用户在引导中选择的角色，将作为参数传递给 AI 分析 prompt，影响"战略意图"和"行动建议"的生成角度，但不改变分析流程本身。
- **规则-2 自有产品信息影响相关性**：用户填写的自有产品信息（目标受众、核心卖点、竞争优势、战略目标）作为 AI 分析的上下文，用于判断竞品变更与用户的相关性和重要性。
- **规则-3 竞品状态转换**：监控中 ↔ 已暂停（用户主动操作）；监控中 → 采集进行中（系统触发采集时自动切换）；采集进行中 → 监控中（采集完成后自动恢复）。
- **规则-4 情报优先级判定**：由 AI 分析管道输出（紧急/中等/低影响），基于变更类型和影响程度判定。
- **规则-5 数据隔离**：用户只能查看和管理自己创建的竞品、收件箱中的报告、以及与自己账户关联的引导配置。
- **规则-6 采集频率**：每个竞品按可配置的默认间隔执行周期性采集（默认值在设计阶段确定）。
- **规则-7 反馈闭环**：标记为"有误"且选择具体错误类型的反馈，应能在后续分析管道中被纳入质量改进（具体机制在设计阶段确定）。

## 6. 验收标准（AC，可测试）

### 6.1 场景 S-001 的 AC

- **AC-001**：新用户首次登录后，自动进入 3 步引导流程；第 1 步展示 6 个角色选项（产品营销经理/产品经理/市场营销经理/创始人/投资人/其他），必须选择一项才能进入下一步。
- **AC-002**：引导第 2 步填写自有产品信息：名称（必填）、URL（必填，需校验格式）、一句话描述、目标受众、核心卖点、竞争优势、战略目标（均为选填）。必填项为空时"下一步"按钮不可点击。
- **AC-003**：引导第 3 步，系统基于第 2 步的 URL 展示竞品推荐列表（若有）。用户可勾选推荐项，也可点击"手动添加"输入竞品名称和 URL。至少添加 1 个竞品（推荐勾选或手动添加均可）后才可完成引导。
- **AC-004**：引导完成后，用户进入主界面，左侧边栏显示已添加的竞品列表（含 Logo 占位、名称、状态），状态初始为"监控中"。
- **AC-005**：竞品添加成功后，系统自动创建首个采集任务并加入异步队列；采集完成后触发生成分析报告，报告出现在收件箱中。

### 6.2 场景 S-002 的 AC

- **AC-006**：收件箱左侧卡片列表每批加载 ≤2s；每条卡片展示标题、关联竞品名称、时间戳、优先级标签（紧急红色/中等橙色/低影响灰色）。
- **AC-007**：收件箱支持按优先级（紧急/中等/低影响）、日期范围（起止日期选择器）、特定竞品（下拉多选）组合筛选；筛选结果即时更新。
- **AC-008**：点击收件箱卡片，右侧展示报告详情，包含：变更摘要列表（每条摘要 ≤3 行）、战略意图分析（1 段文字）、行动建议（按紧急程度分级的条目列表，每条附原因说明）、原始网站跳转链接（新标签页打开）。
- **AC-009**：打开报告详情后，对应卡片从"未读"变为"已读"状态（视觉区分：未读卡片左侧有蓝色标记点）。
- **AC-010**：报告详情页底部展示反馈按钮组（有用 / 有误 / 不重要）；点击"有用"或"不重要"即时确认并显示成功提示；点击"有误"弹出错误类型选择器（8 种：信息不准确、内容不相关、数据过时、信息重复、战略意图有误、遗漏关键变更、噪音过多、源数据不准确），必选一项后提交。
- **AC-011**：空收件箱时展示引导文案和插图，提示"还没有情报报告，系统正在监控中，请稍后查看"或"添加更多竞品以获取情报"。

### 6.3 场景 S-003 的 AC

- **AC-012**：点击左侧边栏"+ 添加竞品"按钮，弹出添加表单：竞品名称（必填）、主域名（必填，需校验格式）、关联链接（选填，可添加多条，上限 10 条）。提交后竞品即刻出现在列表中。
- **AC-013**：点击竞品右侧"暂停监控"按钮（确认弹窗），竞品状态变为"已暂停"，系统不再对该竞品创建新的定时采集任务。
- **AC-014**：暂停的竞品显示"恢复监控"按钮，点击后状态恢复为"监控中"，恢复定时采集。
- **AC-015**：点击"手动刷新"按钮（确认弹窗），竞品状态变为"采集进行中"，系统立即创建采集任务；采集完成后状态恢复，若产生新分析报告则推送至收件箱。
- **AC-016**：点击竞品进入详情页，Overview 面板展示：公司基本面字段（成立时间、公司类型、融资阶段、所在地、员工规模、目标细分市场、领英主页链接——MVP 允许部分字段为空或手动填写） + 最新情报时间线（按时间倒序展示近期分析报告卡片摘要）。

### 6.4 跨场景 AC

- **AC-017**：未登录用户访问任何页面均重定向至登录页；登录后只能访问自己名下的竞品、报告和配置。
- **AC-018**：所有异步操作（添加竞品后首次采集、手动刷新、暂停/恢复）提供加载状态反馈（按钮 loading 态或全局 toast 提示）。
- **AC-019**：所有空数据场景（无竞品、无报告、无筛选结果）展示明确引导文案，不出现空白区域。
- **AC-020**：异常场景（网络错误、服务器 500、采集超时）展示友好错误提示，包含"重试"按钮，不出现白屏或崩溃。

## 7. 异常与边界

- **异常-1 URL 格式校验失败**：引导第 2 步和添加竞品表单中，URL 字段实时校验格式（http/https 协议），格式不合法时字段标红并提示，提交按钮不可点击。
- **异常-2 竞品网站不可达**：采集任务超时或目标网站返回 4xx/5xx，任务标记为"失败"，不生成报告，用户可在竞品列表中看到"采集失败"状态及重试入口。
- **异常-3 关联链接超限**：添加竞品时关联链接超过 10 条上限，添加按钮置灰并提示"已达上限（10 条）"。
- **异常-4 并发采集限流**：后台任务队列对单用户并发采集数设上限，超出部分排队等待，避免资源争抢。
- **异常-5 AI 分析无意义变更**：若网站差异仅为时间戳、随机数等无意义变化，AI 分析管道应标记为"无实质变更"且不生成报告推送收件箱。
- **异常-6 竞品删除确认**：删除竞品为高风险操作，需二次确认弹窗（"删除后将无法恢复该竞品的历史数据和报告，确认删除？"），确认后删除竞品及其关联数据。

## 9. 原型产出判定

- **交互变化结论**：需要原型。全新产品，所有页面均为新建。涉及多处交互设计——3 步引导向导、竞品侧边栏管理、收件箱卡片列表+详情面板布局、反馈按钮组与错误类型选择器弹窗、竞品详情 Overview 面板布局。
- **页面与入口**：全新页面，无复用。主要页面包括：登录/注册页、引导 3 步页、主界面（竞品侧边栏+右侧内容区）、竞品详情页、收件箱页、报告详情页。
- **关键控件/字段与校验**：URL 格式校验、引导步骤进度指示器、竞品状态标签（3 种状态视觉区分）、优先级标签（3 级颜色）、反馈按钮组+错误类型选择弹窗、二次确认弹窗（暂停/删除/手动刷新）。
